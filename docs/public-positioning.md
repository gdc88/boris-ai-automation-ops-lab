# Public technical positioning

This page describes the public technical positioning of the project.

## Short positioning

A portfolio-safe case study showing how self-hosted AI automation can support infrastructure operations, scheduled workflows, evidence capture, and privacy-preserving reporting.

## AI-agent landscape positioning

This project positions Hermes as an **operations layer**, not merely another chat interface. The practical distinction is documented in [Hermes vs Cowork, OpenClaw, and Odysseus](hermes-vs-cowork-openclaw-odysseus.md):

- Hermes is used for repeatable workflows, Telegram delivery, scheduled agents, evidence artifacts, local scripts, and human approval gates.
- Cowork/frontier AI workspaces are useful for high-quality interactive writing, reasoning, and planning.
- OpenClaw-style agents are useful for sandboxed agent-pattern experiments.
- Odysseus/local AI workspaces are useful for private/offline model experiments when hardware and security boundaries are understood.

The professional claim is therefore: designing safe, auditable AI automation for IT/cloud operations — not just using a chatbot.

## Public boundary

- No private credentials or browser sessions.
- No private Telegram, Gmail, Sheets, CV, or tracker data.
- GitHub content should stand on its own as technical evidence and growth history.
